# 打开文件表

两级表：

  • 整个系统一个表，包含与进程无关的信息，每一个文件条目包含：
    • 文件打开次数
      • 变为 0 之后就可以 “关闭文件”，从表中删除该项
    • 文件磁盘位置
    • 访问日期
    • 文件大小
  • 每个进程有一个表，包含与进程有关的信息，每一个文件条目包含：
    • 读写指针
    • 访问权限
    • 指向系统打开文件表的对应文件条目指针

⚠️系统只会在第一次打开一个文件的时候使用其文件名在磁盘上定位，然后创建文件表中的条目，此后就不再需要文件名。

# 文件访问控制表

为每个文件和目录设置一个文件访问控制表。用来管理不同用户、用户组对该文件或目录的访问权限

# 文件逻辑结构

文件访问，或者文件中查找数据的方式，是由文件的逻辑结构决定的，这里讨论的都是有结构文件，也叫做记录式文件，因为这样的文件是由一个以上的记录构成的。

实际上还存在最简单的无结构文件，也叫做流式文件，对该类型文件中记录的访问只能通过穷举搜索。

# 单级目录

实现了按名存取，每个文件名对应唯一一个文件。

缺点：

  • 文件不允许重名
  • 不方便文件共享

# 两级目录

  • 主文件目录 Maste File Directory (MFD)
    • 用户文件目录 User File Directory (UFD)
    • 用户文件目录 User File Directory (UFD)
    • 用户文件目录 User File Directory (UFD)
    • ......

解决了多用户之间文件重名的问题。

# 树形目录

理论上可以实现无限深度的目录层次。

出现了相对路径和绝对路径的概念。

# 无环图目录

比树形目录多了共享子目录，实现了文件、目录共享。

一个文件可以有多个路径，称为别名。例如：

  • Linux 中的软链接、硬链接
  • Windows 的快捷方式

允许了环的出现，实现了自引用的特殊需求。
