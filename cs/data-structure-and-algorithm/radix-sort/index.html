基排序 / 桶排序 / 数字排序

# 原理

需要频繁进行分组重排，（类似于归并）适用于链表

可分为两种：

  • MSD，最高位优先 (Most Significant Digit first)

  • LSD，最低位优先 (Least Significant Digit first)

# 复杂度

设有nnn 个待排序记录，关键字位数为ddd，每位有rrr 种取值。则排序的趟数是ddd; 在每一趟中:

  • 链表初始化的时间复杂度:O(r)O(r)O(r);
  • 分配的时间复杂度:O(n)O(n)O(n);
  • 分配后收集的时间复杂度:O(r)O(r)O(r);

则链式基数排序的时间复杂度为: O(d⋅(n+r))O(d\cdot (n+r))O(d⋅(n+r))

在排序过程中使用的辅助空间是:2r2r2r 个链表指针，nnn 个指针域空间，
实际应用中rrr 远小于nnn，可以得到时间复杂度为：O(dn)O(dn)O(dn)

则空间复杂度为:O(n+r)O(n+r)O(n+r)

  • 当 n 很大，d 比较小小，基数排序可以比归并、快排性能更好

  • 基排序是稳定排序。
